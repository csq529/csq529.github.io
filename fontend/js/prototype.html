# Construcotr构造函数

首先写一个构造函数 Person，构造函数一般为了区分普通函数要求首字母大写。

function Person() {}

# prototype 原型

原型指的就是一个对象，实例“继承”那个对象的属性。在原型上定义的属性，通过“继承”，实例也可拥有了这个属性。“继承”这个行为是在new操作符内部实现的。

先不说实例，原型和构造函数的关系，就是构造函数内部有一个名为prototype的属性，通过这个属性就能访问到原型：

Person是构造函数，Person.prototype就是原型

# instance 实例

有个构造函数，就可以在原型上创建可以“继承”的属性，并通过new操作符创建实例。

比如说Person，要创建一个 person实例，那么使用new操作符就可以实现，并通过 instanceof来检查他们之间的关系：

function Person() {}

let person = new Person()
let res = person instanceof Person // 检查person是否为Person的实例

console.log(res) // true

在原型上定义一个属性，那么实例就可以“继承”这个属性。

function Person() {}
Person.prototype.type = 'object named Person'
let person = new Person() 
let type = person.type // 实例继承的属性
console.log(type) // object named Person

# proto 隐式原型

实例通过__proto访问到原型，所以如果是实例，那么就可以通过这个属性直接访问原型。

所以这两者是等价的

function Person() {}
Person.prototype.type = 'object named Person'

let person = new Person()
let relation = person.__proto__ === Person.prototype

console.log(relation) // true

# constructor 构造函数

既然构造函数通过prototype来访问到原型，那么原型也应该能够通过某种途径访问到构造函数，这个途径就是constructor。

function Person() {}
let person = new Person()
let relation = Person.prototype.constructor === Person
console.log(relation) // true

这里的constructor是原型的一个属性, Contructor指的才是真正的构造函数。

# 实例、构造函数、原型之间的关系

如果实例想要访问构造函数，那么应该是：

function Person() {}
Person.prototype.type = 'object named Person'

let person = new Person()
let relation = person.__proto__.constructor === Person
console.log(relation)

没有从实例直接访问到构造函数的属性或方法。

实例与原型是通过__proto__去访问的。

在读取一个属性的过程中，如果属性在该实例中没有找到，那么就会循着__proto__指定的原型上去寻找，如果找不到，则尝试寻找原型的原型。

function Person() {}
Person.prototype.type = 'type 来自原型'

let person = new Person()
let res = Reflect.ownKeys(person) // 尝试获取到自有属性
console.log(res) // person 对象自身没有属性 []

let type = person.type
console.log(type) // 打印的是原型上的属性   type 来自原型

如果给实例同名属性，可以看到打印出来的属性就指向这个。

function Person() {}
Person.prototype.type = 'type 来自原型'

let person = new Person()
person.type = 'type来自实例的属性'

let res = Reflect.ownKeys(person) // 尝试获取到自有属性
console.log(res) // person 对象自身有type属性 ['type']

let type = person.type // 那么没有必要通过原型链继续向上搜索
console.log(type) // 打印的是实例上的属性    type来自实例的属性

# 原型链

原型同样也可以通过__proto__ 访问到原型的原型，比如这里有个构造函数Person，然后继承前者的有个构造函数 People，然后new People 得到实例p。

当访问p中的一个非自有属性的时候，就会通过__proto__作为桥梁连接起来的一系列原型、原型的原型、原型的原型的原型直到Object构造函数为止。

这个搜索的过程形成的链状关系就是原型链。

function Person () {}
Person.prototype.type = 'type来自Person原型'

function People() {}
Prople.prototype = new Person()

let p = new People()
let relation = [p instanceof Object, p instanceof Person, p instanceof People]
console.log(relation) // [true, true, true]

let type = p.type
console.log(type) // type来自Person的原型

看到null了吗，原型链搜索搜到null为止，搜不到那访问的这个属性就是不存在的。

function Person() {}
Person.prototype.type = 'type 来自Person的原型'

function People() {}
People.prototype = new Person()

let p = new People()
let relation1 = p.__proto__
console.log(relation1) // Person{}

let relation2 = p.__proto__.__proto__
console.log(relation2) // { type: "type 来自Person的原型" }

let relation3 = p.__proto__.__proto__.__proto__
console.log(relation3) // {}

let relation4 = p.__proto__.__proto__.__proto__.__proto__
console.log(relation4) // null

以上，就是原型、原型链、构造函数、实例、null之间的关系。
