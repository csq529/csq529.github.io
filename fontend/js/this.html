# 前言

this的指向在函数定义的时候是确定不了的，只有函数执行的时候才能确定this到底指向谁，实际上this的最终指向的是那个调用它的对象（这样理解其实是不够全面的，下面会解释）。

# this的四种绑定规则

  • # 默认绑定

规则： 在非严格模式下，默认绑定的this指向全局对象，严格模式下this指向undefined。

function fn() {
  console.log(this.a)
}
var a = 1
fn() // 1

function fn2() {
  'use strict'
  console.log(this.a)
}
fn2() // Uncaught TypeError: Cannot read property 'a' of undefined

默认绑定还有一个特殊的地方

function fn() {
  console.log(this.a)
}

function fn2() {
  'use strict'
  fn()
}

var a = 2
fn2() // 2

结论：对于默认绑定来说，决定this绑定对象的是函数体是否处于严格模式，严格模式指向undefined，非全局指向全局对象window

  • # 隐式绑定

规则：函数在调用的位置，是否有上下文对象，如果有，那么this就隐形绑定到这个对象上。

function fn1() {
  console.log(this.a)
}

var a = 'Oops, global'
let obj2 = {
  a: 2,
  foo: fn1
}

let obj1 = {
  a: 22,
  obj2: obj2
}

obj2.foo() // 2 this指向调用函数的对象obj2
obj1.obj2.foo() // 2 this指向最后一层调用函数的对象obj2

// 隐式绑定丢失
let bar = obj2.foo // bar只是一个函数别名，是obj2.foo的一个引用
bar() // 'Oops, global' this 指向全局对象

隐式绑定丢失：实际上就是函数调用时，并没有上下文对象，只是对函数的引用，所以会导致隐形绑定丢失。

同样的问题，还发生在传入回调函数中，这种情况更加常见，并且隐蔽，类似:

test(obj2.foo) // 传入函数的引用，调用时也是没有上下文对象

  • # 显示绑定

就像上面看到的，如果单纯使用隐形绑定肯定没有办法得到期望的绑定的，幸好还可以在某些对象上强制调用函数，从而将this绑定在这个对象上。

规则：可以通过apply、call、bind将函数中的this绑定到指定对象上。

function foo() {
  console.log(this.a)
}
let obj = {
  a: 2
}
foo.call(obj) // 2

如果传入的不是对象，而是一个原始值（字符串、布尔类型、数字类型），来当做this的绑定对象，这个原始值会转换成它的对象形式。

如果把null或者undefined作为this的绑定对象传入call/apply/bind，这些值会在调用时被忽略，实际应用的是默认绑定规则。

  • # new绑定

new的时候会做哪些事情：

  1. 创建一个全新的对象；
  2. 这个新对象会被执行[[Prototype]]连接；
  3. 这个新对象会绑定到函数调用的this；
  4. 如果函数没有返回其他对象，那么new表达式中的函数调用会自动返回这个对象。

规则：使用构造调用的时候，this会自动绑定到new期间创建的对象上。

function Fn() {
  this.a = 1
}
let bar = Fn foo()
console.log(bar.a) // 1

这里之所以对象bar可以点出函数Fn里面的a是因为new关键字可以改变this的指向，将这个this指向对象bar,为什么说bar是对象，因为用了new关键字就是创建一个对象实例。

当this碰到return时

// 例1
function fn() {
  this.a = 1
  return {}
}
var bar = new fn
console.log(bar.a) // undefined

// 例2
function fn1() {
  this.a = 1;
  return function() {}
}

var bar1 = new fn1
console.log(bar1.a) // undefined

// 例3
function fn2() {
  this.a = 1;
  return 2
}
var bar2 = new fn2
console.log(bar2.a) // 1

// 例4
function fn3() {
  this.a = 1;
  return undefined
}

var bar3 = new fn3
console.log(bar3.a) // 1

结论：如果返回值是一个对象，那么this指向的就是那个返回的对象，如果返回值不是一个对象那么this还是指向函数的实例。

function fn() {
  this.a = 1;
  return undefined
}
var bar = new fn
console.log(bar) // fn { a: 1 }

还有一点就是虽然null也是对象，但在这里this还是指向那个函数的实例，因为null比较特殊。

function fn() {
  this.a = 1
  return null
}

var bar = new fn
console.log(bar.a) // 1

# this四种绑定规则的优先级

如果在某个调用位置应用了多条规则，如何确定哪条规则生效？

obj.foo.call(obj2) // this指向obj2 显式绑定比隐式绑定优先级高
new obj.foo() // this指向new新创建的对象 new绑定比隐式绑定优先级高

显式绑定和new绑定无法直接比较（会报错），默认绑定是不应用其他规则之后的兜底绑定所以优先级最低，最后结果是：

显式绑定 > 隐式绑定 > 默认绑定

new 绑定 > 隐式绑定 > 默认绑定

# 箭头函数的this指向不会使用上述的四条规则：

function foo() {
  return () => {
    console.log(this.a)
  }
}
let obj1 = {
  a: 1
}

let obj2 = {
  a: 22
}

let bar = foo.call(obj1) // foo this指向obj1
bar.call(obj2) // 输入2 这里执行箭头函数 并试图绑定this指向到obj2

从以上可以得出，箭头函数的this规则：

1.箭头函数this继承于它外面第一个不是箭头函数的函数this指向。

2.箭头函数的this一旦绑定的上下文，就不会被任何代码改变。
