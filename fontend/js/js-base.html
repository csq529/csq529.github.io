    # 防抖和节流

    防抖：单位时间内，频繁的触发事件，只执行最后一次

    • 应用场景：输入框搜索

    • 思路：利用定时器，每次触发先清掉以前的定时器（从新开始）

        function debounce(fn, delay = 300) {
        let timer = null
        return function(...args) {
            clearTimeout(timer)
            timer = setTimeout(() => {
                fn.call(this, ...args)
            }, delay)
        }
    }

    使用: debounce(() => {}, 1000)

    节流：单位时间内，频繁的触发事件，只执行一次

    • 应用场景：表单提交、鼠标滑动、resize事件、scroll事件
    • 思路：利用定时器，等定时器执行完毕，才开启定时器（不要打断）
        function throttle(fn, delay = 300) {
        let timer = null
        return function(...args) {
            if (time === null) {
                timer = setTimeout(() => {
                    fn.call(this, ...args)
                    timer = null
                }, delay)
            }
        }
    }

    使用：window.addEventListener('scroll', throttle(function () {
            console.log('scroll')
        }, 500))

    可以使用lodash库，利用里面的debounce（防抖）和throttle（节流）

    # 原型及原型链

    原型：每个对象都有一个prototype属性，称它为原型（对象），原型上可以放一些属性和方法，共享给实例对象，原型可以继承

    原型链：每个对象都有对象都有一个__proto__属性，这个属性指向它的原型对象，原型对象也是对象，也有一个__proto__属性，指向原型对象的原型对象，这样一层一层形成的链式结构就是原型链，最顶层找不到就是null

    # 事件循环（EventLoop）1

    • JS是单线程，防止代码阻塞，把任务分为同步和异步
    • 同步代码给JS引擎执行，异步代码交给宿主环境（浏览器、Node）
    • 同步代码放入执行栈，异步代码等待时机成熟送入任务队列排队
    • 执行栈执行完毕，回去任务队列查看是否有异步代码，有就送到执行栈执行，反复循环查看执行这个过程就是事件循环（EventLoop）

    宏任务和微任务

    • 宏任务：宿主环境发起（浏览器、Node） 常见：script 事件、网络请求、定时器、动画、IO

    • 微任务：JS引擎发起的任务 Promise本身是同步，then/catch的回调函数是异步的

      process.nextTick(Node)，Promise.then() Promise.catch()

      async/await

    • 执行过程：

      • 同步任务
      • 微任务的异步代码
      • 宏任务的异步代码
